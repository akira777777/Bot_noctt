# Анализ и План реализации Bot_noct

## Обзор проекта

**Bot_noct** - это Telegram bot наастроенный на **Telegraf** для управления трафиком каналов, обработки каталога, приёма лидов и административных конвераций.

### Текущая архитектура

```
src/
├── bot.js                 # Главная точка входа, регистрация команд
├── config/
│   └── env.js              # Конфигурация окружения
├── db/
│   ├── sqlite.js           # База данных SQLite
│   └── migrations/         # Миграции схемы
├── domain/
│   └── lead-status.js      # Состояния лидов
├── handlers/
│   ├── admin.js           # Обработчики для администратора
│   └── client.js          # Обработчики для клиентов
├── repositories/         # Доступ к данным
├── services/             # Бизнес-логика
├── ui/                   # Компоненты интерфейса
├── utils/                # Утилиты
├── web/                  # Веб-интерфейс
└── webapp/              # Мини-приложение
```

### Ключевые компоненты

1. **Bot Core** (`bot.js`) - сервер на Telegraf с регистрацией команд
2. **Services Layer** - бизнес-логика (лиды, админ, конверации)
3. **Handlers** - обработчики сообщений для разных ролей
4. **Repositories** - абстракция доступа к данным
5. **UI Components** - клавиатурные элементы и сообщения
6. **Web Interface** - административный панель контроля
7. **Mini App** - Веб-приложение для контекстного входа

## Критические функциональные требования

### Бизнес-логика

1. **Lead Management** - многошаговый процесс приёма заявок
2. **Conversation Management** - система диалогов с клиентами
3. **Admin Interface** - интерфейс для управления лидами и диалогами
4. **Catalog Management** - хранилище продуктов и статистики
5. **Source Tracking** - определение источника трафика

### Технические требования

2. **Session Management** - многошаговый процесс приёма заявок
